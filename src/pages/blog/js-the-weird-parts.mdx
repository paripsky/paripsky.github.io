1. Adding 0.1 + 0.2 will give the wrong result
```js
const result = 0.1 + 0.2;
console.log(result); // 0.30000000000000004
```

2. The same happens when multiplying 0.56 by 100
```js
const result = 0.56 * 100;
console.log(result); // 56.00000000000001
```

3. The maximum safe integer in javascript is 2^53 - 1
```js
const result = 2 ** 53 === 2 ** 53 + 1;
console.log(result); // true
```
Due to 2^53 being the maximum safe integer in javascript, doing math operations on numbers that are larger will give inaccurate results.

4. Making a banana üçå
```js
('ba' + + 'a' + 'a').toLowerCase() // banana
```

`+ + 'a'` is the same as doing `Number('a')` which results in NaN. Adding 'ba' in the beginning and 'a' in the end gives us 'baNaNa', then we use .toLowerCase() to get 'banana'.
